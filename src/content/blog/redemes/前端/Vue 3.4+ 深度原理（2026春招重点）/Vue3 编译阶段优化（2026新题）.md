## 一、静态提升（Static Hoisting）

### 1. 静态节点提升
```javascript
// 编译前模板
<template>
  <div>
    <span>静态文本</span>
    <p>{{ dynamicData }}</p>
  </div>
</template>

// 编译后代码
const _hoisted_1 = /*#__PURE__*/ _createVNode("span", null, "静态文本", -1 /* HOISTED */);

function render() {
  return _createVNode("div", null, [
    _hoisted_1, // 静态节点复用
    _createVNode("p", null, _toDisplayString(dynamicData), 1 /* TEXT */)
  ]);
}
```

### 2. 静态属性提升
```javascript
// 编译前
<button class="btn btn-primary" :class="{ active: isActive }">点击</button>

// 编译后
const _hoisted_1 = { class: "btn btn-primary" };

function render() {
  return _createVNode("button", 
    _mergeProps(_hoisted_1, { class: { active: isActive } }), 
    "点击", 
    2 /* CLASS */
  );
}
```

## 二、Patch Flags 优化

### 1. Patch Flags 类型
```javascript
// Patch Flags 枚举
export const enum PatchFlags {
  TEXT = 1,                    // 动态文本节点
  CLASS = 1 << 1,              // 动态 class
  STYLE = 1 << 2,              // 动态 style
  PROPS = 1 << 3,              // 动态属性（不包括 class/style）
  FULL_PROPS = 1 << 4,         // 动态 key，需要完整 diff
  HYDRATE_EVENTS = 1 << 5,     // 带事件监听器的节点
  STABLE_FRAGMENT = 1 << 6,    // 子节点顺序不会改变的 Fragment
  KEYED_FRAGMENT = 1 << 7,     // 带 key 的 Fragment
  UNKEYED_FRAGMENT = 1 << 8,   // 不带 key 的 Fragment
  NEED_PATCH = 1 << 9,         // 非 props 的更新
  DYNAMIC_SLOTS = 1 << 10,     // 动态 slot
  HOISTED = -1,                // 静态提升节点
  BAIL = -2                    // 表示 diff 算法应该结束
}
```

### 2. 运行时优化
```javascript
// 根据 Patch Flags 进行针对性更新
function patchElement(n1, n2, parentComponent) {
  const el = n2.el = n1.el;
  const oldProps = n1.props || EMPTY_OBJ;
  const newProps = n2.props || EMPTY_OBJ;
  
  // 根据 Patch Flags 决定更新策略
  const { patchFlag } = n2;
  
  if (patchFlag > 0) {
    if (patchFlag & PatchFlags.CLASS) {
      // 只更新 class
      if (oldProps.class !== newProps.class) {
        hostPatchProp(el, 'class', null, newProps.class);
      }
    }
    
    if (patchFlag & PatchFlags.STYLE) {
      // 只更新 style
      hostPatchProp(el, 'style', oldProps.style, newProps.style);
    }
    
    // 其他类型更新...
  } else {
    // 完整 diff
    // Vue2 的更新方式
  }
}
```

## 三、Tree Shaking 支持

### 1. 按需导入优化
```javascript
// Vue3 支持 Tree Shaking
import { createApp, ref } from 'vue';

// 只打包使用的 API
const app = createApp({
  setup() {
    const count = ref(0);
    return { count };
  }
});

// 未使用的 API 不会被包含在最终包中
```

### 2. 编译时 Tree Shaking
```javascript
// 编译时检测未使用的组件
<template>
  <div>
    <UsedComponent />
    <!-- UnusedComponent 不会被包含 -->
  </div>
</template>

<script>
import UsedComponent from './UsedComponent.vue';
import UnusedComponent from './UnusedComponent.vue'; // 会被 Tree Shaking

export default {
  components: { UsedComponent }
};
</script>
```

Vue3 的编译阶段优化使得应用性能得到显著提升。
